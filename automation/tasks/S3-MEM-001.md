# S3-MEM-001

## Goal
Wire confidence decay into memory recall ranking.

## Scope
- In the SQLite memory backend's recall/search path (`src/memory/sqlite.zig`), after fetching results, apply `decay.decayConfidence()` to each result using the record's created_at timestamp.
- Use `decay.defaultParamsForKind()` with `.raw` kind as default (since TypedRecord fields aren't in the schema yet).
- Sort or re-rank results by decayed confidence (multiplied with existing relevance score).
- Add `recencyScore()` as a secondary ranking signal.
- Add tests with mocked timestamps showing that older records rank lower than newer ones with equal keyword relevance.

## Acceptance
- `decayConfidence()` called during recall in SQLite backend
- `recencyScore()` used as tiebreaker
- Results are re-ranked by decayed score
- Tests verify ordering: recent record beats old record at equal relevance
- All existing memory tests still pass
