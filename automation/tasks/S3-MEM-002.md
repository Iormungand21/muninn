# S3-MEM-002

## Goal
Wire hybrid vector+keyword search end-to-end in the SQLite memory backend.

## Scope
- In `src/memory/sqlite.zig`, update the recall/search path to:
  1. Run FTS5 keyword search (existing)
  2. If an embedding provider is configured, compute query embedding and run vector cosine similarity search against stored embeddings
  3. Call `vector.hybridMerge()` to combine both result sets with configurable weights
- In the memory store path, if an embedding provider is configured, compute and cache the embedding for new entries using `embeddings.cacheEmbedding()`.
- Read vector_weight and keyword_weight from memory config.
- Add tests verifying that hybridMerge is called when embeddings are available, and that keyword-only search still works when no embedding provider is configured.

## Acceptance
- Store path computes and caches embeddings when provider available
- Recall path runs hybrid search when embeddings exist
- Falls back to keyword-only when no embedding provider
- Config weights are respected
- All existing memory tests still pass
