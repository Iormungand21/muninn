# S3-SEC-001

## Goal
Wire secret scope enforcement into the secret retrieval path.

## Scope
- In `src/security/secrets.zig` (or wherever `SecretStore.get()` is called), add a call to `scope.isSecretAccessible()` before returning a secret value.
- Accept workspace_id and channel_id as parameters in the retrieval path.
- If the secret's scope restricts access for the given workspace/channel, return null or an appropriate error.
- Wire `findWorkspacePolicy()` + resolve helpers into the policy evaluation path in `src/security/policy.zig` so per-workspace autonomy overrides take effect.
- Add tests verifying that a workspace-scoped secret is denied to a non-allowed workspace.

## Acceptance
- `isSecretAccessible()` is called on the hot path, not just available as a helper
- `findWorkspacePolicy()` resolve functions are called during policy evaluation
- Tests cover: global secret accessible everywhere, workspace-scoped secret denied to wrong workspace, channel-scoped secret denied to wrong channel
- All existing tests still pass
