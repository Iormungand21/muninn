# S3-SEC-002

## Goal
Implement real Landlock syscall bindings for the Landlock sandbox backend.

## Scope
- In `src/security/landlock.zig`, add Zig `@cImport` or inline syscall wrappers for:
  - `landlock_create_ruleset` (syscall 444)
  - `landlock_add_rule` (syscall 445)
  - `landlock_restrict_self` (syscall 446)
- Define `landlock_ruleset_attr`, `landlock_path_beneath_attr` structs.
- Implement `applyLandlock()` that creates a ruleset allowing read/write only within the workspace directory, then restricts the current process.
- Update `wrapCommand()` to call `applyLandlock()` before exec, or document why the current pattern (apply to self) is correct.
- Guard all syscalls behind Linux OS check; return error on non-Linux.
- Add tests that verify struct layouts and error paths (actual enforcement needs root/capable process).

## Acceptance
- Landlock structs and syscall numbers defined
- `applyLandlock()` function exists and makes the three syscalls in order
- `isAvailable()` checks kernel version support (>= 5.13)
- Tests compile and pass on all platforms (with graceful skip on non-Linux)
- No behavioral change on non-Linux
