# M5-GW-001

## Goal
Expand /health endpoint with component-level status detail.

## Scope
- In `src/gateway.zig`, update the `/health` endpoint handler to:
  - Return a JSON object with component health from `HealthRegistry` (instead of just `"ok"`).
  - Include: memory backend status, provider connectivity, channel health, daemon uptime.
  - Format: `{"status": "ok", "components": {"memory": "ok", "provider": "ok", "channels": {"discord": "ok", "slack": "degraded"}}}`.
  - Return HTTP 200 when all components healthy, 503 when any critical component is down.
- Wire the existing `HealthRegistry` snapshot into the handler.
- Add tests for healthy/degraded/unhealthy response formatting.

## Acceptance
- `/health` returns component-level JSON
- HTTP status reflects overall health (200 vs 503)
- Uses existing HealthRegistry data
- Tests cover all health states
- All existing gateway tests still pass
