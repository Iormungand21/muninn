# M4-GW-001

## Goal
Replace gateway subprocess fallback with graceful error handling.

## Scope
- In `src/gateway.zig`, in the `processIncomingMessage()` function (around line 464-497):
  - Remove the subprocess spawn fallback path.
  - When `session_mgr` is unavailable, return a proper JSON error response: `{"error": "service_unavailable", "message": "Session manager not initialized"}` with HTTP 503 status.
  - Log a warning when this path is hit.
- Update the `/ready` endpoint to report not-ready when session_mgr is null.
- Add tests verifying the 503 response when session_mgr is unavailable.

## Acceptance
- No subprocess spawned in gateway message handling
- 503 response with JSON error body when session_mgr unavailable
- `/ready` reflects session_mgr availability
- Tests cover the error path
- All existing gateway tests still pass
